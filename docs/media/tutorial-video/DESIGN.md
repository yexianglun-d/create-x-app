# create-x-app-cli 宣传视频视觉系统

## 风格定位

15 秒 GitHub 开源宣传片采用克制的 Apple 产品片段气质：深色空间、低对比金属层次、缓慢镜头运动和大留白。画面重点不是录屏复现 CLI，而是把终端、模板矩阵和插件生态重构为有纵深的产品空间，让“脚手架基础设施”这个概念在视觉上成立。

## 色彩

- 背景石墨黑：`#07090D`，用于整片基础空间。
- 深钛灰：`#111820`，用于终端、卡片和抽象 GitHub 面板。
- 雾白：`#F4F7F8`，用于主标题和关键命令。
- 银灰：`#AEB8BC`，用于副标题、说明文字和辅助信息。
- 冷杉绿：`#8FB9A8`，用于成功状态、节点连接和温和强调。
- 香槟金：`#D6C7A1`，用于镜头光带和高级金属边缘。

## 字体

- 中文主字体：`PingFang SC`, `Hiragino Sans GB`, `MiSans`, `Microsoft YaHei`, sans-serif。
- 命令与数据字体：`SFMono-Regular`, `JetBrains Mono`, `Cascadia Mono`, `Menlo`, monospace。
- 标题使用 700-800 字重，正文使用 350-400 字重；深色背景下保持充足行高。

## 布局原则

- 画布固定为 1920x1080，内容安全区不少于 120px。
- 终端窗口、能力卡片、插件节点使用轻微 3D-like 分层，不做真实录屏。
- 每屏文字少而明确，优先大标题和一条副标题，功能信息以短标签呈现。
- 装饰粒子少量、固定位置、弱对比，不抢内容层级。

## 动效原则

- 镜头使用慢速 parallax、柔和淡入和细微纵深位移。
- 场景之间用柔和光带滑过和空间层级平移，避免跳切。
- GSAP easing 以 `power3.out`、`power2.inOut`、`expo.out` 为主。
- 所有 timeline 必须 `paused: true` 并注册到 `window.__timelines`。

## 禁止事项

- 不做普通 README 录屏，不做 TikTok 式快切。
- 不使用廉价蓝紫霓虹渐变、强描边发光或过量粒子。
- 不出现右上角水印。
- 不堆砌长段功能说明，不让文字溢出画布。
