# create-x-app-cli 宣传视频视觉系统 v2

## 创意方向

新版不再采用“终端 + 卡片”的静态拼贴，而是做成一个连续的空间化产品片段：一条 CLI 命令点亮工程核心，能力沿轨道展开，插件生态接入，最后收束到 GitHub 开源发布。画面需要更像 Apple 发布会里的产品系统展示，而不是 README 截图或普通教程录屏。

## 视觉关键词

- 深色钛金属空间
- 低对比玻璃界面
- 一枚中心工程核心
- 轨道、层板、生态节点
- 缓慢镜头推进与焦点转移
- 少文字、强留白、强层次

## 色彩

- 深空黑：`#05070A`，全片背景基底。
- 钛灰：`#10161C`，终端、面板和层板底色。
- 雾白：`#F3F6F4`，主标题和关键命令。
- 冰灰：`#AAB6B7`，副标题和说明。
- 鼠尾草绿：`#8DB7A4`，成功状态、能力轨道、插件连接。
- 柔金：`#D8C79D`，品牌强调、光线滑过、最终 CTA。
- 暗铜：`#8F7960`，极弱暖色环境光。

## 字体

- 中文主字体：`PingFang SC`, `Hiragino Sans GB`, `Microsoft YaHei`, sans-serif。
- 命令字体：`JetBrains Mono`, `Cascadia Mono`, `Menlo`, monospace。
- 主标题使用 760-820 字重；正文使用 350-420 字重。
- 深色背景中正文行高保持 1.45 以上，避免字体发糊。

## 构图规则

- 1920x1080 固定画布，主要内容安全区 130px 以上。
- 每幕至少有两处视觉焦点：主文案 + 产品装置/终端/生态。
- 主视觉组件采用 3D-like transform，不能像网页卡片平铺。
- 场景之间使用光线滑过 + 焦点模糊过渡，避免硬切。
- UI 文案控制在短句，功能信息尽量以标签、轨道和状态呈现。

## GSAP 动效规则

- 使用 `gsap.timeline({ paused: true, defaults: ... })`，并注册到 `window.__timelines`。
- 使用 labels 管理四个段落：`hook`、`infra`、`ecosystem`、`finale`。
- 使用 `from()` 做每幕入口，使用 `fromTo()` 做光线转场。
- 不使用 `repeat: -1`，不使用 `Math.random()`，不动画 `display/visibility`。
- 不让所有元素从同一方向进入；每幕使用不同节奏和 easing。

## 禁止事项

- 不做录屏式 CLI 演示。
- 不做蓝紫霓虹、廉价强发光、过量粒子。
- 不堆满功能说明，不做信息表格。
- 不出现水印。
- 不把项目宣传片做成 TikTok 快节奏短视频。
