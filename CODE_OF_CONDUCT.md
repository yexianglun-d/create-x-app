# 社区行为准则

`create-x-app` 希望保持专业、友好、直接且高质量的协作氛围。所有参与者在 Issue、Pull Request、代码评审、讨论和社区插件协作中都应遵守本准则。

## 我们鼓励的行为

- 尊重不同经验水平和技术背景的贡献者。
- 围绕事实、复现步骤、代码和可验证证据讨论问题。
- 对不同方案给出清晰理由，而不是简单否定。
- 在代码评审中指出具体文件、具体风险和可执行建议。
- 主动补充文档、示例和验证步骤，降低后续维护成本。

## 不可接受的行为

- 人身攻击、侮辱、威胁、骚扰或歧视性表达。
- 发布他人隐私、密钥、token、内部链接或未授权信息。
- 在安全问题修复前公开可利用细节。
- 反复刷屏、恶意顶帖、无关营销或垃圾链接。
- 明知会破坏用户环境仍引导他人执行危险命令。

## 讨论原则

- 技术争议以复现、测试、文档和代码为准。
- 如果一个问题缺少复现信息，维护者可以要求补充后再继续处理。
- 如果一个 PR 范围过大，维护者可以要求拆分。
- 如果一个模板或插件引入过多维护成本，维护者可以拒绝合并并说明原因。

## 执行方式

维护者可以根据情况采取以下措施：

- 要求修改措辞或补充信息。
- 编辑或隐藏不当内容。
- 关闭偏离主题或无法复现的问题。
- 暂停或拒绝继续参与项目协作。

如果你遇到违反本准则的行为，请通过 GitHub Issue 或维护者可见的渠道反馈，并提供相关链接和上下文。
