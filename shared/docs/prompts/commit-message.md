# Commit Message Prompt

Write a Conventional Commit message for the staged changes.

Rules:

- Use one of: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `chore`, `ci`, `revert`.
- Keep the subject under 100 characters.
- Prefer an imperative, concrete subject.
- Add a short body only when the change has migration notes or non-obvious tradeoffs.
